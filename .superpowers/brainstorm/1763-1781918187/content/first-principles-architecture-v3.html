第一性原理重构：Quiet Neurosymbolic Investing Copilot

不是“把 Agent、RAG、PyReason 拼在一起”，而是从保护注意力、保留决策主权、提高证据质量重新推导系统。

PROBLEM ESSENCE

唯一核心任务

持续观察你的私人持仓，只在新证据可能改变 thesis 或风险暴露时打扰你，并且让你看得懂为什么。

它减少研究和监控负担，但永远不接管 conviction，也不执行交易。

不可再分解的 Ground Truths

1 · 市场真相不完整
缺少证据不等于事实为假；系统必须允许 unknown。
2 · LLM 会误读和脑补
它可以提取、关联、解释，不能单独建立事实或决定打扰。
3 · 你的注意力是稀缺资源
提醒本身有成本；“保持沉默”必须是一等输出。
4 · Conviction 属于你
AI 只能提出带证据的 diff，不能偷偷修改长期立场。
5 · 证据会过时且互相冲突
必须记录发生时间、观察时间、来源、置信度和反证。
6 · 无评估就没有可信提醒
Eval、审计与回放从 MVP 开始，不是最后装饰。

原计划中被挑战的假设

原假设判决第一性原理修正
Eval 放到 C / v4丢弃MVP 就记录每次候选事件、触发/沉默原因、重复率和人工反馈；C 只负责扩大基准集。
Agent 自主决定定时提醒丢弃定时管线是确定性的；Agent 只服务手动研究和问答。两条执行路径物理分离。
PyReason 是统一规则引擎修改简单阈值继续用 Python policy engine；PyReason 专注跨实体、时间、冲突证据的推理，并置于可替换接口后。
新闻是系统的核心对象丢弃核心对象改为 evidence claim / event；一条新闻可能包含多条证据，多篇新闻也可能是同一事件。
200 日线是产品核心修改它只是一个可解释传感器插件；真正核心是 thesis/risk change detection。
越聪明就越应该自动记忆修改分成 working inference 与 confirmed memory；前者可自动生成，后者必须经你确认。

重新推导后的三阶段

A · TRUSTWORTHY MVP

Personal Policy + Quiet Alerts

  • 持仓、Conviction Card、风险宪法
  • EOD 价格、200 日线、SEC/公司一手事件
  • Evidence Ledger：每条事实带 provenance
  • 确定性 policy engine：阈值、冷却、每日预算
  • 事件指纹去重，不急着上 pgvector
  • 先 shadow mode，再真实推送
  • Discord + 审计日志 + 基础 Eval
→
B · NEUROSYMBOLIC COPILOT

Evidence Graph + PyReason

  • 只读研究 Agent，与定时提醒管线分离
  • LLM 生成带引用的 candidate claims
  • Evidence Gate 检查来源、时效、冲突与交叉认证
  • 公司、指标、产业因子、thesis 组成时间证据图
  • PyReason 推导支持/削弱/未知及完整 proof trace
  • 向你提出 Conviction Card diff，必须确认
  • PyReason spike 必须证明优于简单规则才扩大使用
→
C · SCALE & SCIENCE

Platform + Evaluation

  • Embedding + pgvector 语义事件聚类
  • 来源冗余、Telegram 等多渠道
  • 历史回放与规则版本化
  • 新闻/事件 precision、recall
  • 幻觉、引用正确性、越界行为 Eval
  • 提醒价值、漏报率、用户注意力成本
最终执行边界：两条路径，绝不混在一起

Scheduled Alert Pipeline

数据 → 事实验证 → 规则/PyReason 输出 → 必要性门控 → Discord

每一步可回放；LLM 只能把已通过门控的证据说成人话。

Interactive Research Agent

你的问题 → 自主调只读工具 → 搜集与质疑证据 → 回答 / 提出 diff

可以聪明地探索，但不能修改 confirmed memory、规则或 conviction。

最重要的删减

MVP 不做全网新闻理解、不做向量数据库、不做多 Agent、不做自动 conviction、不做交易执行。先证明：连续 30 天，它的沉默和提醒都值得信任。
